🧠 一句話心智模型：JavaScript = 一個放值的盒子系統 + 一條一次只做一件事的執行緒。

你做的事只有三種：①把值存起來（變數、物件、陣列）、②寫規則處理值（運算子、條件、迴圈、函式）、③對外界起反應（DOM 事件、網路請求）。第三種是非同步的來源，也是新手 90% 的困惑所在。

建議的學習順序（也就是本篇的章節）：值與型別 → 流程控制 → 函式與閉包 → 物件與陣列 → DOM 與事件 → 非同步 → 模組與常見坑。不要跳章，尤其第 3 章（函式）沒懂，第 6 章（非同步）一定卡。

★ 學 JS 最有效的方法是邊看邊打。按 F12 打開瀏覽器 Console，本篇每一段程式碼都可以直接貼進去按 Enter 看結果。看十遍不如自己跑一遍。

0 · 開始之前 — JavaScript 是什麼、在哪裡跑

0.1 JavaScript 到底是什麼

JavaScript 是一種直譯式程式語言，最初只在瀏覽器裡跑，用來讓網頁「動起來」。今天它三個地方都能跑：

環境用來做什麼額外能力
瀏覽器網頁互動、前端整個生態（React、Vue…）DOM、fetch、localStorage
Node.js後端 API、CLI 工具、建置腳本檔案系統、網路伺服器
其他桌面 App（Electron）、手機（React Native）依平台而定

★ 語言核心到處一樣（第 1–4、6–7 章），只有第 5 章 DOM 是瀏覽器專屬。所以先把核心學好，換環境不用重學。

0.2 十秒開始寫第一行 — 不用裝任何東西

在這個網頁上按 F12（Mac：Cmd + Option + I）→ 切到 Console 分頁 → 打字 → 按 Enter。就這樣，你已經在寫 JavaScript 了。

// 貼進 Console 按 Enter，會跳出一個對話框
console.log("Hello, JavaScript!");

// 它也是一台計算機
2 + 3 * 4      // → 14

要寫成網頁的一部分時，用 <script> 標籤把 JS 檔掛進 HTML。永遠加 defer——這樣瀏覽器會先把 HTML 讀完再跑 JS，不會發生「元素還不存在就想操作它」的錯。

<!DOCTYPE html>
<html>
<head>
  <!-- defer：HTML 解析完才執行，且不阻擋畫面 -->
  <script src="main.js" defer></script>
</head>
<body>
  <h1 id="title">Hello</h1>
</body>
</html>

三個一定要學會的偵錯工具（新手最常忽略、但省下最多時間）：

console.log()
把任何值印到 Console。卡住時第一件事就是印出來看，不要用猜的。
console.table(陣列) 印成表格更好讀
Console 的紅字
錯誤訊息會直接告訴你「哪一行、什麼錯」。把它讀完，別看到紅字就慌。
最常見：xxx is not defined／undefined
debugger
在程式碼裡寫一行 debugger;，開著 F12 時會停在那裡，讓你逐行看每個變數的值。
比 console.log 更快找出邏輯錯誤

1 · 值與型別 — 資料長什麼樣

1.1 變數：只用 const 和 let，忘掉 var

變數就是幫一個值取名字，之後用名字來指它。

const name = "Michael";   // 常數：之後不會重新指派 → 預設都用這個
let count = 0;             // 變數：之後會改它的值
count = count + 1;         // ✅ let 可以改

name = "Someone";          // ❌ TypeError: Assignment to constant variable
var old = 1;               // ❌ 舊寫法，作用域規則會咬人，別再用

取捨很簡單：一律先寫 const，只有當你確定要重新指派時才改成 let。這不是潔癖——const 讓讀程式的人（包含三個月後的你）一眼知道「這個名字不會變」。

constletvar（別用）
能重新指派否是是
作用域區塊 { }區塊 { }整個函式（容易出錯）
宣告前使用報錯（好事）報錯（好事）得到 undefined（爛事）

★ const obj = {…} 是指「obj 這個名字不會指向別的東西」，不是「裡面的內容不能改」。obj.age = 30 完全合法。這點下一節的圖會講清楚。

1.2 八種型別，但你天天用的只有五種

// ── 原始型別（primitive）：值本身 ──
const n = 42;              // number  （整數、小數都是它，沒有 int/float 之分）
const s = "hi";            // string  （單雙引號都行，樣板字串用反引號）
const b = true;            // boolean （true / false）
let   u;                   // undefined（宣告了但沒給值）
const e = null;            // null    （刻意表示「這裡是空的」）

// ── 物件型別（object）：一包東西 ──
const obj  = { a: 1 };     // object
const arr  = [1, 2, 3];    // array 也是 object
const fn   = () => {};     // function 也是 object

// 想知道型別就用 typeof
typeof 42        // "number"
typeof "hi"      // "string"
typeof null      // "object" ← 語言的歷史 bug，記住就好
Array.isArray(arr) // true  ← 判斷陣列要用這個

字串好用招式——樣板字串（template literal），用反引號 `，可以直接把變數插進去、也能換行：

const user = "Michael", age = 30;

// ❌ 舊寫法，加號串一堆很難讀
"Hi " + user + ", you are " + age + " years old."

// ✅ 樣板字串
`Hi ${user}, you are ${age} years old.`
`還能直接
換行`

1.3 最關鍵的一張圖：原始型別 vs 物件型別

這是新手第一個大坑：複製一個數字和複製一個物件，行為完全不同。

// 原始型別：值複製
let a = 1, b = a;  b = 2;
console.log(a);            // 1  ← 不受影響

// 物件：參考複製
let x = { n: 1 }, y = x;  y.n = 2;
console.log(x.n);          // 2  ← 跟著變了！

// ✅ 真的想要一份新的，用展開運算子淺拷貝
let z = { ...x };  z.n = 99;
console.log(x.n);          // 2  ← 這次不受影響了

// 巢狀物件要深拷貝（現代瀏覽器內建）
const deep = structuredClone(x);

★ 這條規則會一路影響到 React 的 state 更新、陣列排序、函式參數傳遞。現在花五分鐘想通，之後省十小時 debug。

1.4 運算子與真假值 — === 永遠優先於 ==

JavaScript 會偷偷幫你轉型別（type coercion），這是它最惡名昭彰的地方。解法很單純：比較一律用 ===。

"1" == 1       // true  ← == 會先轉型再比，結果很難預測
0 == false     // true
null == undefined // true

"1" === 1      // false ← === 型別不同直接 false，可預測
0 === false    // false

NaN === NaN     // false ← 唯一例外，判斷要用 Number.isNaN(x)

Falsy 值只有六個，其他一切都是 truthy。背起來，因為 if (x) 每天都在用：

// 這六個放進 if 都是 false：
false, 0, "", null, undefined, NaN

// ⚠ 這些「看起來空」的都是 true！
"0"  []  {}  // 空字串是 false，但空陣列/空物件是 true

三個現代必學運算子，能砍掉大量 if 判斷：

const user = { profile: { name: "Michael" }, score: 0 };

// ?. 選擇性串連：中間是 null/undefined 就回 undefined，不會爆炸
user.address?.city        // undefined（不報錯）
user.address.city          // ❌ TypeError

// ?? 空值合併：只有 null/undefined 才用預設值
user.score ?? 100         // 0   ← 0 是有效值，保留
user.score || 100         // 100 ← ❌ 0 被當成 falsy 蓋掉了，經典 bug

// 三元運算子：一行的 if/else
const label = user.score > 60 ? "pass" : "fail";
互動：== 與 === 到底差在哪
點一組看 == 與 === 的差別，順便看看轉型後到底變成什麼。

2 · 流程控制 — 讓程式會分岔、會重複

2.1 條件判斷

const score = 75;

if (score >= 90) {
  console.log("A");
} else if (score >= 60) {
  console.log("Pass");
} else {
  console.log("Fail");
}

// 分支多又都是「等於某個值」時，switch 比較好讀
switch (action) {
  case "add":    doAdd();  break;   // ⚠ 忘了 break 會往下繼續執行
  case "remove": doRemove(); break;
  default:        doNothing();
}

★ 巢狀 if 超過兩層就該重寫。技巧叫 early return：把不合格的情況先擋掉 return，主邏輯就能留在最外層，讀起來輕鬆很多。

2.2 迴圈：知道三種就夠

const fruits = ["apple", "banana", "cherry"];

// ① for...of ── 逐一走過陣列，90% 的情況用它就對了
for (const f of fruits) console.log(f);

// ② for ── 需要索引、或要控制步進 / 反向時才用
for (let i = 0; i < fruits.length; i++) console.log(i, fruits[i]);

// ③ while ── 次數未知，靠條件決定何時停
let n = 1;
while (n < 100) n = n * 2;

// 走物件的鍵值對
for (const [key, value] of Object.entries({a:1, b:2})) console.log(key, value);

// 兩個控制字：
break;     // 整個迴圈直接停
continue;  // 跳過這一輪，繼續下一輪

★ 實務上處理陣列時，更常用的其實是 map / filter / reduce（第 4 章），而不是手寫迴圈。但迴圈是基礎，一定要會。

3 · 函式與閉包 — 整個語言的核心

函式就是把一段規則包起來、取個名字、之後餵不同的值進去重複使用。這章沒懂，第 6 章非同步一定卡。

3.1 三種寫法，用途不同

// ① 函式宣告 ── 會被提升（hoisting），寫在下面也能先呼叫
function add(a, b) { return a + b; }

// ② 函式表達式 ── 存進變數，必須先宣告才能用
const sub = function (a, b) { return a - b; };

// ③ 箭頭函式 ── 現代預設寫法，短、且 this 行為單純
const mul = (a, b) => a * b;        // 單一表達式 → 自動 return
const greet = name => `Hi ${name}`;  // 單一參數可省括號
const log = (msg) => {              // 有大括號就要自己 return
  console.log(msg);
  return msg.length;
};

參數的兩個實用招式：

// 預設參數：沒傳就用預設值
function greet(name, greeting = "Hello") { return `${greeting}, ${name}!`; }
greet("Michael");            // "Hello, Michael!"

// 其餘參數：把剩下的通通收成陣列
function sum(...nums) { return nums.reduce((a, b) => a + b, 0); }
sum(1, 2, 3, 4);            // 10

3.2 作用域與閉包 — 面試必考，但概念其實很直覺

作用域（scope）：一個變數「看得到的範圍」。內層看得到外層，外層看不到內層。

const APP = "site";             // 全域

function makeCounter() {
  let count = 0;               // makeCounter 的區域變數
  return function increment() {
    count++;                   // ← 記得外層的 count，這就是閉包
    return count;
  };
}

const c1 = makeCounter();
c1(); c1();                    // 1, 2
const c2 = makeCounter();
c2();                          // 1  ← 全新一份，跟 c1 無關

console.log(count);            // ❌ ReferenceError：外面看不到
互動：兩個計數器各自記得自己的 count
0
counter 1
0
counter 2
兩顆按鈕呼叫的是同一個 makeCounter 產生的兩個獨立閉包——各自記著自己的 count，永遠不會互相污染。

閉包在真實世界的用處：做出「私有變數」（外面改不到）、事件處理器記住當時的資料、以及第 6 章的非同步 callback 能存取當初的上下文。

3.3 函式是「值」— 高階函式

在 JS 裡函式本身就是一個值，可以當參數傳、也可以被回傳。這條規則撐起了整個現代 JS 的寫法。

// 把函式當參數傳進去（callback）
[1, 2, 3].map(n => n * 2);          // [2, 4, 6]
button.addEventListener("click", () => alert("hi"));

// 回傳一個函式（工廠）
const multiplier = factor => n => n * factor;
const double = multiplier(2);
double(5);                        // 10

4 · 物件與陣列 — 真實資料長這樣

4.1 物件：用名字存一包相關的值

const user = {
  name: "Michael",
  age: 30,
  tags: ["js", "react"],
  greet() { return `Hi, ${this.name}`; }   // 物件裡的函式叫「方法」
};

user.name;            // "Michael"   點記法（首選）
user["name"];        // "Michael"   鍵名是變數時才用中括號
user.email;           // undefined   讀不存在的鍵不會報錯
user.email = "a@b.c"; // 直接新增
delete user.age;      // 刪除
"name" in user;      // true  判斷有沒有這個鍵

Object.keys(user);    // ["name","tags","greet","email"]
Object.entries(user); // [["name","Michael"], …] 適合拿去跑迴圈

解構與展開——現代 JS 幾乎每一行都會用到，一定要熟：

// 解構：一次把要的欄位拆出來
const { name, age = 18 } = user;        // age 沒有就用 18
const [first, second] = ["a", "b"];      // 陣列也能解構

// 函式參數解構：呼叫端不用管參數順序，超好用
function draw({ x = 0, y = 0, color = "red" }) { /* … */ }
draw({ color: "blue", x: 10 });

// 展開：複製 + 合併（後面的蓋掉前面的）
const updated = { ...user, age: 31 };   // 產生新物件，原本的 user 不動
const merged  = [...arr1, ...arr2];

4.2 陣列方法 — 學會這七個，迴圈可以少寫一半

關鍵分野：會不會改到原陣列。改到原陣列（mutating）的方法是很多 bug 的來源。

方法做什麼回傳改到原陣列？
map每個元素轉換成新的新陣列（等長）否
filter留下符合條件的新陣列（較短）否
find找第一個符合的那個元素 / undefined否
reduce把整個陣列摺成一個值任何東西否
some / every有沒有一個 / 是不是全部符合boolean否
forEach逐一執行副作用（不轉換）undefined否
push/pop/sort/reverse/splice增刪改排序視方法而定是 ⚠
const nums = [1, 2, 3, 4, 5];

nums.map(n => n * 2);              // [2,4,6,8,10]
nums.filter(n => n % 2 === 0);     // [2,4]
nums.find(n => n > 3);             // 4
nums.reduce((sum, n) => sum + n, 0); // 15  （0 是初始值，別忘了給）
nums.some(n => n > 4);             // true
nums.every(n => n > 0);            // true

// 可以串起來，像一條加工流水線 —— 這是 JS 最舒服的地方
const users = [
  { name: "A", age: 17, active: true  },
  { name: "B", age: 25, active: true  },
  { name: "C", age: 30, active: false }
];
const names = users
  .filter(u => u.active && u.age >= 18)   // 先篩
  .map(u => u.name)                        // 再轉換
  .join(", ");                          // "B"

// ⚠ sort 會改到原陣列，而且預設是「字串排序」！
[10, 9, 1].sort()                  // [1, 10, 9] ← 不是你要的
[10, 9, 1].toSorted((a,b) => a-b)  // [1, 9, 10] ← 給比較函式，且不改原本
互動：陣列方法流水線
[1,2,3,4,5,6] 經過：
目前結果 [1,2,3,4,5,6]
連續點按鈕就是在串接方法——每一步都產生新的陣列，原本那個從頭到尾沒被改過。

5 · DOM 與事件 — 讓網頁真的動起來

到這裡為止都是「語言核心」，換到 Node.js 也一樣。這一章是瀏覽器專屬：DOM 是瀏覽器把 HTML 解析成的一棵樹，JS 透過它讀寫畫面。

5.1 選元素、改元素

// 選：只要記住這兩個，用 CSS 選擇器語法
const el   = document.querySelector("#title");        // 第一個符合的
const all  = document.querySelectorAll(".item");       // 全部（NodeList）

// 改內容
el.textContent = "新標題";      // ✅ 純文字，安全，首選
el.innerHTML = userInput;       // ⚠ 會解析成 HTML，放使用者輸入 = XSS 漏洞

// 改樣式：優先切 class，而不是直接寫 style
el.classList.add("active");
el.classList.toggle("open");
el.style.color = "red";         // 少用，樣式該留在 CSS

// 改屬性 / 讀資料
el.setAttribute("aria-label", "關閉");
el.dataset.id;                    // 對應 HTML 的 data-id="…"

// 新增元素
const li = document.createElement("li");
li.textContent = "新項目";
document.querySelector("#list").appendChild(li);
li.remove();                      // 刪掉自己

5.2 事件：接住使用者的動作

button.addEventListener("click", (e) => {
  console.log("被點了", e.target);
});

// 表單送出：一定要擋掉預設的整頁重新載入
form.addEventListener("submit", (e) => {
  e.preventDefault();
  const data = Object.fromEntries(new FormData(form));
  console.log(data);
});

// 常見事件：click / input / change / submit / keydown / scroll

事件委派（delegation）——這招每個前端都在用：不要在 100 顆按鈕上各掛一個監聽，掛在共同的父層一個就好。連之後才新增的元素也自動有效。

// ❌ 每顆各掛一個：元素多就慢，而且新增的元素沒監聽
document.querySelectorAll(".item").forEach(el => el.addEventListener("click", …));

// ✅ 父層掛一個，靠冒泡接住全部
list.addEventListener("click", (e) => {
  const item = e.target.closest(".item");   // 往上找最近的 .item
  if (!item) return;                        // 點到空白處就忽略
  console.log("點了", item.dataset.id);
});
互動：事件冒泡 — 一次點擊，三層都收到
外層 div（掛了監聽）
內層 div（掛了監聽）
點按鈕看事件怎麼一路往上冒泡；點內層或外層的空白處，比較差在哪。

6 · 非同步 — 新手最大的坎

JavaScript 是單執行緒：一次只做一件事。所以「等網路回應」這種慢事情不能卡著等，必須交給瀏覽器去做、等它好了再回頭處理。這就是非同步的全部動機。

6.1 先看懂 Event Loop，其他都好說

console.log("1");
setTimeout(() => console.log("2"), 0);   // 就算是 0 毫秒
Promise.resolve().then(() => console.log("P"));
console.log("3");

// 輸出順序：1 → 3 → P → 2
// 同步先跑完 → 微任務（Promise）→ 巨任務（setTimeout）
互動：執行順序播放 — 為什麼是 1 3 P 2
下面會照真實順序，一行一行印出來。

6.2 Promise：非同步結果的「憑證」

Promise 是一個代表「未來會有結果」的物件，三種狀態：pending（等待中）→ fulfilled（成功）或 rejected（失敗）。狀態一旦確定就不會再變。

// 幾乎不用自己 new Promise，通常是用別人回傳的（例如 fetch）
fetch("/api/user")
  .then(res => res.json())        // 成功 → 接下一棒
  .then(data => console.log(data))
  .catch(err => console.error(err)) // 任何一棒失敗都會掉到這
  .finally(() => setLoading(false)); // 成功失敗都會跑

6.3 async / await：把非同步寫得像同步（首選寫法）

// 同一件事，用 async/await 讀起來像一行一行的同步程式碼
async function loadUser(id) {
  try {
    const res  = await fetch(`/api/user/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`); // ⚠ fetch 只有網路斷才 reject，404 不會！
    const user = await res.json();
    return user;
  } catch (err) {
    console.error("載入失敗:", err);
    return null;
  }
}

// 呼叫端：async 函式永遠回傳 Promise，所以還是要 await 或 .then
const user = await loadUser(1);

新手最常犯的三個錯：

// ❌ 錯 1：忘了 await，拿到的是 Promise 物件不是資料
const user = loadUser(1);
console.log(user.name);   // undefined —— user 是 Promise{}

// ❌ 錯 2：明明可以同時跑，卻排隊等（3 秒變 1 秒的差別）
const a = await fetchA();   // 等 1 秒
const b = await fetchB();   // 再等 1 秒
const c = await fetchC();   // 再等 1 秒 → 共 3 秒

// ✅ 互不相依就一起發，Promise.all 等全部好
const [a, b, c] = await Promise.all([fetchA(), fetchB(), fetchC()]); // 共 1 秒

// ❌ 錯 3：在 forEach 裡 await —— 完全沒有等待效果
items.forEach(async (i) => { await save(i); });  // 迴圈不會等
for (const i of items) { await save(i); }        // ✅ 要循序就用 for...of
await Promise.all(items.map(i => save(i)));      // ✅ 要並行就用 all
互動：循序 await vs Promise.all（各 3 個 1 秒的請求）
進度
0.0秒
三個互不相依的請求，用 Promise.all 一起發，時間從 3 秒變 1 秒。

7 · 模組化與常見的坑

7.1 模組：把程式拆成多個檔案

// ── math.js ──
export function add(a, b) { return a + b; }   // 具名匯出（可多個）
export default class Calculator { }              // 預設匯出（每檔最多一個）

// ── main.js ──
import Calculator, { add } from "./math.js";    // 副檔名不能省

// ── index.html ── 一定要加 type="module"
<script src="main.js" type="module"></script>

★ type="module" 的檔案自動是嚴格模式、自動 defer、且有自己的作用域（不會污染全域）。另外模組不能用 file:// 直接開，需要一個本機伺服器，最快的方式：npx serve。

7.2 六個一定會踩到的坑

1. this 在箭頭函式裡不一樣
一般函式的 this 看「誰呼叫它」；箭頭函式沒有自己的 this，直接沿用外層。物件的方法用一般函式，callback 用箭頭函式，就幾乎不會出錯。
obj.method(){…} ✓　callback: () => {} ✓
2. 浮點數不精確
0.1 + 0.2 !== 0.3（是 0.30000000000000004）。這不是 JS 的 bug，所有語言的浮點數都這樣。
錢的計算用「整數分」存，別用小數
3. 非同步結果拿太早
在 .then / await 外面讀資料一定是 undefined。資料在哪拿到，就在哪用。
忘記 await 是最常見的單一錯誤
4. 陣列方法改到原陣列
sort／reverse／splice 會就地改。要保留原本就先複製，或用新的 toSorted／toReversed。
[...arr].sort() 或 arr.toSorted()
5. innerHTML 放使用者輸入
等於把 XSS 攻擊的門打開。純文字一律用 textContent。
textContent 是安全的預設
6. 相等比較用 ==
隱式轉型的規則沒人記得住，直接用 ===。唯一例外：x == null 可同時判 null 和 undefined。
開 ESLint 的 eqeqeq 規則自動擋
